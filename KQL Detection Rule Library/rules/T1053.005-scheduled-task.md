# T1053.005 — Scheduled Task/Job: Scheduled Task

**Tactic:** Persistence  
**Severity:** Medium  
**Data Source:** `DeviceProcessEvents` (Defender for Endpoint)  
**Query Frequency:** Every 1 hour | **Lookback:** 24 hours

---

## Detection Rationale

Adversaries use `schtasks.exe` to establish persistence by scheduling malicious executables or scripts to run automatically. CLI-based task creation (as opposed to GUI) is the dominant method used in post-exploitation frameworks and is a reliable persistence indicator when properly filtered.

---

## KQL Query

```kql
DeviceProcessEvents
| where TimeGenerated > ago(1d)
| where FileName =~ "schtasks.exe"
| where ProcessCommandLine has "/create"
| where InitiatingProcessFileName !in~ ("msiexec.exe", "setup.exe", "install.exe")
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine, InitiatingProcessFileName
```

---

## Tuning Log

| Field | Detail |
|-------|--------|
| Baseline Noise | ~25 alerts/day |
| Tuning Period | 7 business days |
| Final FP Rate | ~2–3 alerts/day |

**Exclusions Applied:**
- Excluded `msiexec.exe`, `setup.exe`, `install.exe` as initiating processes
- Excluded Veeam Backup agent scheduled task creation
- Allowlisted known patch management task names via Sentinel watchlist

**Analyst Notes:** Review task name and command path. Random-looking names or tasks pointing to `%TEMP%`, `%APPDATA%`, or user-writable paths are high-priority.

---

## Response Actions

1. Review full task definition: path, trigger, run-as account
2. Check if task binary or script exists and is signed
3. Correlate with other persistence indicators (registry runs, services)
4. Delete malicious task and hunt for related artifacts
