# T1059.001 — Command and Scripting Interpreter: PowerShell Encoded Command

**Tactic:** Execution  
**Severity:** High  
**Data Source:** `DeviceProcessEvents` (Defender for Endpoint)  
**Query Frequency:** Every 1 hour | **Lookback:** 24 hours

---

## Detection Rationale

Encoded PowerShell commands (`-EncodedCommand`, `-enc`, `-ec`) are a primary obfuscation technique used to hide malicious payloads from basic script logging and string-matching defenses. Legitimate use exists but is heavily concentrated in known management tooling that can be excluded.

---

## KQL Query

```kql
DeviceProcessEvents
| where TimeGenerated > ago(1d)
| where FileName =~ "powershell.exe" or FileName =~ "pwsh.exe"
| where ProcessCommandLine has_any ("-EncodedCommand", "-enc", "-ec")
| where ProcessCommandLine !has "MicrosoftEdge"
| project TimeGenerated, DeviceName, AccountName, ProcessCommandLine, InitiatingProcessFileName
```

**Exclusions Applied:**
- Excluded SCCM client processes where `CCMExec.exe` is the initiating process
- Excluded Microsoft Intune Management Extension process tree
- Excluded signed Microsoft binaries as initiating process
- Excluded known monitoring agent hashes via watchlist

**Analyst Notes:** Stack with Event ID 4104 (Script Block Logging) and AMSI telemetry for compound detection. User-context encoded commands with unknown parent processes are highest priority.

---

## Response Actions

1. Decode the Base64 payload — use CyberChef or PowerShell: `[System.Text.Encoding]::Unicode.GetString([System.Convert]::FromBase64String('<payload>'))`
2. Review decoded content for download cradles, C2 callbacks, or privilege escalation
3. Check initiating process — unexpected parents (Word, Excel, browser) are critical
4. Isolate endpoint if decoded payload is confirmed malicious
