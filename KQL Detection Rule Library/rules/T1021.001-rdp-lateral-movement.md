# T1021.001 — Remote Services: Remote Desktop Protocol (Lateral Movement)

**Tactic:** Lateral Movement  
**Severity:** High  
**Data Source:** `DeviceNetworkEvents`, `DeviceInfo` (Defender for Endpoint)  
**Query Frequency:** Every 1 hour | **Lookback:** 24 hours

---

## Detection Rationale

RDP connections originating from standard workstations (not jump servers or admin machines) are anomalous and suggest an adversary is using RDP for lateral movement after initial compromise. Legitimate helpdesk and admin RDP activity is concentrated in known source systems that can be excluded.

---

## KQL Query

```kql
DeviceNetworkEvents
| where TimeGenerated > ago(1d)
| where RemotePort == 3389
| where InitiatingProcessFileName !in~ ("mstsc.exe", "msrdc.exe")
    or DeviceName !startswith "JUMP"
| join kind=leftouter (
    DeviceInfo | project DeviceName, DeviceType
) on DeviceName
| where DeviceType == "Workstation"
| project TimeGenerated, DeviceName, RemoteIP, RemotePort, InitiatingProcessFileName
```

**Exclusions Applied:**
- Excluded IT helpdesk workstations (`HELP-*` naming convention)
- Excluded connections to lab and test servers
- Excluded connections during approved maintenance windows

**Analyst Notes:** Workstations RDP-ing to domain controllers are critical priority. Correlate with AD group membership to identify privilege context.

---

## Response Actions

1. Identify source workstation — who is logged in? Is it expected?
2. Check destination — is it a server, DC, or another workstation?
3. Review Event ID 4624 (logon) on destination for successful authentication
4. If unauthorized — isolate both source and destination, investigate for broader compromise
